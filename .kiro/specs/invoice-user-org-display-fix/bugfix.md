# Bugfix Requirements Document

## Introduction

HTML invoices generated by the system currently display minimal user and organization information, showing only a UUID for the user account and no organization ID at all. This makes invoices difficult to identify and lacks user-friendly information that customers expect to see on billing documents. This bugfix addresses the missing display of user name, email, organization ID, and organization name in generated invoice HTML.

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN an invoice is generated with userId THEN the system displays only the UUID as "Account ID" in the invoice HTML

1.2 WHEN an invoice is generated with organizationId THEN the system does not display the organization ID anywhere in the invoice HTML

1.3 WHEN an invoice is generated THEN the system does not display user name in the invoice HTML

1.4 WHEN an invoice is generated THEN the system does not display user email in the invoice HTML

1.5 WHEN invoice generation is called from routes THEN the system does not fetch user data (name, email) before passing to generateInvoiceHTML

1.6 WHEN invoice generation is called from routes THEN the system does not pass organization name to generateInvoiceHTML despite it being available in some queries

### Expected Behavior (Correct)

2.1 WHEN an invoice is generated with userId THEN the system SHALL fetch and display the user's name in the invoice HTML

2.2 WHEN an invoice is generated with userId THEN the system SHALL fetch and display the user's email in the invoice HTML

2.3 WHEN an invoice is generated with organizationId THEN the system SHALL display the organization ID in the invoice HTML

2.4 WHEN an invoice is generated with organizationId THEN the system SHALL display the organization name in the invoice HTML

2.5 WHEN invoice generation is called from routes THEN the system SHALL fetch user data (name, email) from the users table before generating the invoice

2.6 WHEN invoice generation is called from routes THEN the system SHALL fetch organization data (name) from the organizations table before generating the invoice

2.7 WHEN user or organization data is not available THEN the system SHALL display appropriate fallback text instead of crashing

### Unchanged Behavior (Regression Prevention)

3.1 WHEN an invoice is generated without userId THEN the system SHALL CONTINUE TO generate the invoice without displaying user information

3.2 WHEN an invoice is generated THEN the system SHALL CONTINUE TO display invoice number, date, items, totals, and status correctly

3.3 WHEN an invoice is generated THEN the system SHALL CONTINUE TO apply theme colors and company branding correctly

3.4 WHEN an invoice is generated THEN the system SHALL CONTINUE TO store the invoice in the billing_invoices table with correct data

3.5 WHEN an invoice HTML is generated THEN the system SHALL CONTINUE TO be printable and maintain proper formatting

3.6 WHEN invoice generation fails to fetch optional data THEN the system SHALL CONTINUE TO generate the invoice with available information
